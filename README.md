# outcome-visualizer

Outputs are what you deliver. Outcomes are what changes because of it. This repo hosts a tool related to modeling this difference. And outcome includes value. And output includes cost. When making outcome-based contracts, the risks related to costs are not managed by detailed plans, but continuous visibility.

## Getting Started

Open `index.html` in your browser to use the app. No server required.

## Features

- **Create Outcomes**: Define what changes you want to achieve (includes value description)
- **Create Outputs**: Define what you deliver (includes cost description)
- **Link Outputs to Outcomes**: Map your deliverables to the changes they create
- **Visual Mapping**: See all outcome-output relationships at a glance
- **Data Persistence**: All data stored in browser localStorage
- **Export/Import**: Save and restore your data as JSON files

## Usage

1. Add outcomes (the changes you want to achieve)
2. Add outputs (what you deliver) and optionally link them to outcomes
3. View the mapping visualization to see how outputs contribute to outcomes
4. Export your data to save it or share with others
5. Import previously exported data to restore your work
